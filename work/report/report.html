We have analyzed these websites:
Websoft, HKR, Kristianstad
The main thing we noticed when analyzing the data was that when loading any of the three websites without cache, the amount of data transferred was significantly larger. As expected, all the loading times were lowered when in cache, but on the smaller websites, the difference was very small. On the largest website, Kristianstad.se, the time was more than halfed. On Pagespeed, we noticed that both HKR and Kristianstad had a pretty bad mobile score. Websoft on the other hand, got 100/100, both on mobile and desktop! According to pagespeed, some suggestions to improve the speed on the webpage are:

*Compress text
*Remove resources that are blocking the rendering
*Send images in a more modern format
*Remove unused CSS-code
When analyzing Kristianstad without cache, the most time saving suggestion was to pre-load resources. This tells us that caching the website can be the most important performance booster of them all.

TIL: We learned about different tools that are used to measure performance on websites. We also learned about the most common problems with slow websites.
